# ✅ Deploy Checklist — รอบ hardening (12 มิ.ย. 2026)

runbook เดียวจบ · ทำ **เรียงลำดับบนลงล่าง** · แต่ละสเตปมี "ทำอะไร → เช็กว่าผ่านยังไง"
ลำดับเหตุผล: รีสตาร์ทให้โหลดโค้ดใหม่ → ทดสอบว่าทำงานจริง → ค่อย commit → เก็บกวาด cloud

---

## ☐ 0. ก่อนเริ่ม (เครื่อง Windows ที่มีโค้ด)
- ปิดไฟล์ `สรุปความแม่น_อาจารย์วาสินี.docx` ใน Word (กัน lock)
- ไม่มีหน้าต่าง git อื่นค้างอยู่

## ☐ 1. รีสตาร์ทแอปทุกเครื่องที่เปิดค้าง
- ทุกเครื่อง (มุ้กก + หัวหน้า + พยาบาล A–D): กด `Ctrl+C` ที่หน้าต่างที่รัน Streamlit แล้วรันใหม่
  ```
  streamlit run main_or_app.py
  ```
- **เช็กผ่าน:** หน้าแรกโหลดได้ ไม่มี error แดง

## ☐ 2. ทดสอบ 2 เครื่อง (A กับ B) — หัวใจของรอบนี้
ทำตามนี้ทีละข้อ ดูว่าอีกเครื่องเห็นภายใน ~30 วิ:

- ☐ **A** กด "เข้าห้อง" เคสหนึ่ง → **B** เห็นเคสนั้นเป็น "กำลังผ่า" ใน 30 วิ
- ☐ **B** กด ✏️ แก้เวลา/ย้ายห้อง เคสหนึ่ง → **A** เห็นค่าใหม่ใน 30 วิ *(จุดที่เคยหายเงียบ)*
- ☐ **A** กด "จำหน่าย" เคส → **B** เห็นสถานะอัปเดต *(จุดที่เคยหายเงียบ)*
- ☐ **A** กด "ล้างกระดานวันนี้" → **B** บอร์ดว่างใน 30 วิ (เคสไม่คืนชีพ)
- ☐ ลองตัดเน็ต A ชั่วคราวแล้วกดปุ่ม → ขึ้น ⚠️ "บอร์ดกลางออฟไลน์" (ไม่โกหกว่าซิงก์แล้ว)

**ถ้าข้อไหนไม่ผ่าน → หยุด แจ้งผมก่อน commit**

## ☐ 3. Commit จากเครื่อง Windows
```powershell
cd C:\Dev\main_OR_app
.\commit_all.ps1
```
- สคริปต์จะ **py_compile 17 ไฟล์ก่อน** — ถ้าไม่ผ่านจะหยุด ไม่ commit (ด่านกัน mount ตัดไฟล์)
- **เช็กผ่าน:** เห็น `[OK] เสร็จแล้ว` + `git log -1` โผล่ commit ใหม่
- ถ้าขึ้น ExecutionPolicy ติด: `powershell -ExecutionPolicy Bypass -File .\commit_all.ps1`

## ☐ 4. ล้าง snapshot บอร์ดเก่าบน Supabase (ครั้งเดียว)
```powershell
python purge_board_state.py
```
- ล้าง `board_state_*` เก่าที่สะสมไว้ก่อนมีระบบ retention (ของใหม่ลบเองอัตโนมัติ >7 วัน)
- **เช็กผ่าน:** ขึ้นจำนวน key ที่ลบ ไม่มี error

## ☐ 5. Rebackfill AI (หน้า Maintenance ในแอป)
- เข้าหน้า Maintenance → กดปุ่ม **Rebackfill AI predictions** 1 รอบ
- ทำให้ `ai_model_ver` สะอาด (ล้างแถวเก่า ml_v7/NULL) → หน้า "ความแม่น AI" แสดงเฉพาะผลโมเดลจริง
- **เช็กผ่าน:** ขึ้นจำนวนแถวที่อัปเดต + หน้าความแม่น AI เปิดได้ไม่ error

## ⚙️ (ไม่บังคับ) ปรับ secrets.toml
- ลบบรรทัด `snapshot_keep_pii` ได้ (ตายแล้ว)
- เพิ่ม `db_pool_max = 20` เฉพาะถ้าอยากปรับขนาด pool (ดีฟอลต์ 20 อยู่แล้ว)
- `database_url` ควรลงท้าย `?sslmode=require` (ถ้าไม่ใส่ ระบบเติมให้อัตโนมัติ)

## 🔙 ถ้าต้องย้อนกลับ (rollback)
```powershell
git log --oneline -5        # หา commit ก่อนหน้า
git revert <hash>           # ย้อน commit นี้ (ปลอดภัยกว่า reset)
```

## 📌 สถานะหลังทำครบ
- ระบบพร้อม 10+ ผู้ใช้พร้อมกัน (pool retry/keepalives · เฉลี่ย ~0.6/20 connection)
- ไม่มีชื่อเต็ม/HN เต็มออกนอกเครื่องทางใดอีก (mask + escape + ตัด procnote)
- เลข MAE สอดคล้องทุกที่ (deploy 41.8 · ตารางเทียบ 41.9 matched cohort)
- เหลือ limitation เชิงวิชาการที่เขียนในบทที่ 5 (merge ไม่มี CAS แท้ · cold-path ~20 ฟังก์ชัน) — ดู HARDENING_2026-06-12 §3
