# ตรวจ UX/UI + แผนปรับ OR Flow — 11 มิ.ย. 2026

> บริบท: ผู้ใช้จริง = พยาบาลห้องผ่าตัด + หัวหน้าหอ (หลายคนไม่ถนัดคอม) · อุปกรณ์หลัก = คอม/โน้ตบุ๊ก (เมาส์)
> หลักการแก้: production app ที่ใช้จริง → ทำทีละสไลซ์ที่ "ปลอดภัย + เห็นผล" แล้วให้เปิดดูบนแอปจริงก่อนไปต่อ

## สิ่งที่ของเดิมทำดีอยู่แล้ว (เก็บไว้ ไม่แตะ)

- บอร์ดเป็นตารางแถวเดียว · **แถวไม่ขยับเมื่อสถานะเปลี่ยน** (หาเคสเดิมได้ทันที — สำคัญมากต่อ workflow)
- นาฬิกานับเดินสดฝังในแถว (รอ/กำลังผ่า) · สลับ "เกินเวลา" อัตโนมัติ ไม่ต้องรีเฟรช
- ไฟฉุกเฉินกะพริบ (pulse) · ปุ่มย้อนกลับกันกดพลาด · กันเข้าห้องซ้ำ
- สีสถานะ **สื่อความหมายอยู่แล้ว** (เขียว=กำลังผ่า เหลือง=รอ แดง=เกิน) — ตรงกับ mockup พอดี

## ปัญหา UX/UI ที่ตรวจเจอ (เรียงตามผลกระทบต่อพยาบาล)

| # | ปัญหา | ผลต่อผู้ใช้ |
|---|---|---|
| 1 | **ตัวหนังสือเล็ก 11–13px ทั้งบอร์ด** | พยาบาลนั่งห่างจอ/สายตาไม่ดีต้องเพ่ง — งานเร่งด่วนยิ่งพลาดง่าย |
| 2 | **ระบบดีไซน์ 3 ชุดตีกัน** (Sarabun+gradient / Inter flat / admin gradient) | หน้าตาไม่เป็นอันเดียว ดูไม่น่าเชื่อถือ |
| 3 | **อิโมจิเยอะ ไม่สม่ำเสมอ** (🤖🏥⚙️🔴🟡) | สื่อความหมายไม่คงที่ ดูไม่โปร |
| 4 | **สีตกแต่งไม่สื่อความหมาย** (ไล่ม่วง/ชมพู/ฟ้าเพื่อสวย) | สีไม่ช่วยอ่านสถานะ = noise |
| 5 | **หน้าบริหารยาวมาก** (เรียง section ซ้อน ~11 บล็อก) | ต้องเลื่อนเยอะกว่าจะเจอที่ต้องการ |

## ✅ สไลซ์ที่ 1 — ทำแล้ววันนี้ (ปลอดภัย เห็นผลชัด)

**A. บอร์ดติดตาม (หน้าที่ใช้ทุกวัน) — `tracking_board.py`**
ดันสเกลฟอนต์ทั้งบอร์ดให้สม่ำเสมอและใหญ่ขึ้น (แก้ค่าล้วน ไม่แตะ logic/นาฬิกาสด):

| องค์ประกอบ | เดิม | ใหม่ |
|---|---|---|
| ชื่อผู้ป่วย | 13px | **15px** |
| รหัสห้อง | 12px | **14px** |
| หัตถการ · แพทย์ / เวลานัด | 11.5–12px | **13px** |
| ชิปสถานะ | 11px | **12.5px** (เพิ่ม weight) |
| เซลล์เวลา / นาฬิกาสด | 11.5px | **13px** |
| หัวตาราง · HN · ป้ายเล็ก | 10–11px | **12–12.5px** (ไม่ต่ำกว่า 11.5) |

— ครบทั้ง 3 ตัว render (แถวสถิต + iframe รอผ่าตัด + iframe กำลังผ่า) · ดันความสูง iframe 54→60px กันบรรทัดตัด · ฟอนต์ iframe เปลี่ยนนำด้วย IBM Plex Sans Thai ให้ตรงธีม

**B. ธีมกลาง — `ui_theme.py`**
เพิ่มชุดคลาสชิปสถานะกลาง `.or-chip` (wait/hold/inor/over/post/done = สี=ความหมาย) ให้หน้าอื่นเรียกใช้แทน inline color ในสไลซ์ถัดไป · ดันฟอนต์ caption/pill ที่เล็กไป (11–12 → 12.5px)

**C. เลิก gradient banner เด่น → พื้นเรียบ (flat clinical)**
- หน้าตั้งค่า: หัวข้อไล่สีฟ้า → พื้นขาว + แถบซ้ายน้ำเงิน (`main_or_app.py`)
- หน้าบริหาร: แบนเนอร์ "เคสในเวลา" (ไล่เขียว) + "เคสนอกเวลา" (ไล่ชมพู) → พื้นขาว + แถบสีซ้าย (`main_or_admin.py`)

ทุกไฟล์ compile ผ่าน · ไม่แตะ logic การทำงาน/นาฬิกา/ปุ่ม — แค่ค่าฟอนต์/สีพื้น

## ▶️ สไลซ์ที่ 2 — เสนอทำต่อ (หลังมุ้กกเปิดดูสไลซ์ 1 บนแอปจริง)

1. **หน้าบริหาร — ลดความยาว** ด้วย progressive disclosure: KPI band + การ์ดห้อง (เห็นปราดเดียว) อยู่บนสุด · section ที่เหลือ (รับเวร/นอกเวลา/ภาระงาน) พับใน expander
2. **เปลี่ยนอิโมจิเป็นชุดไอคอนเดียว** ทั้งแอป (ตอนนี้ทำได้เฉพาะที่เป็น HTML — Streamlit ปุ่มยังต้องใช้ emoji หรือไอคอน unicode ชุดเดียว)
3. **การ์ด KPI หน้าบริหาร** ใช้คลาส `.kpi-card` กลางให้เหมือนกันทุกจุด (ตอนนี้บางที่ยัง inline)
4. **รวมหัวข้อ section** ที่ยัง inline gradient อ่อนๆ (group-header variants) เข้าธีมเดียว
