## นโยบายใหม่ (ตามคำสั่งมุ้กก)

แสดงชื่อ-นามสกุล/HN บน dashboard **ได้** แต่ปกปิดบางส่วน**เสมอ ทุกที่ ไม่มี flag ยกเว้น**:

| ข้อมูล | รูปแบบที่แสดง/เก็บ | ตัวอย่าง |
|---|---|---|
| ชื่อผู้ป่วย | คำนำหน้า + ชื่อต้น + นามสกุลย่อ | นางสาว ญาณิศา สุทธิบูรณ์ → `น.ส. ญาณิศา ส.` |
| HN | 4 ตัวท้าย | 563009567 → `…9567` |

ใช้ทั้ง 3 ชั้น: จอแสดงผล · ไฟล์ snapshot local · payload บอร์ดกลางบน Supabase
(flag `snapshot_keep_pii` ถูก**ยกเลิกถาวร** — ตั้งค่าไว้ก็ไม่มีผล)

## อัปเดตรอบ 2 (บ่ายวันเดียวกัน) — รองรับยศ/คำนำหน้าพิเศษ

ปัญหาที่มุ้กกพบบนจอจริง: ผู้ป่วยมียศ (ร.ต.อ. / พล.ต.ต. / ว่าที่ ร.ต.) ถูกตัวตัดชื่อ
กลายเป็น "ร.ต.อ. ม." (ยศ + นามสกุลย่อ — **ชื่อจริงหายทั้งคำ**) เพราะโค้ดเดิมรู้จักแค่
นาย/นาง/น.ส./ด.ช./ด.ญ. แล้วเข้าใจว่ายศคือชื่อต้น

แก้ใน `mask_patient_name` (main_or_db.py): **ยศ/คำนำหน้า (กี่คำก็ได้) + ชื่อต้นเต็ม + นามสกุลย่อ**

| Input | เดิม (ผิด) | ใหม่ |
|---|---|---|
| ร.ต.อ. มานพ สันติสุข | ร.ต.อ. ส. | `ร.ต.อ. มานพ ส.` |
| ว่าที่ ร.ต. พงษ์ศักดิ์ ตันติกุล | ว่าที่ ต. | `ว่าที่ ร.ต. พงษ์ศักดิ์ ต.` |
| พลตำรวจตรี เกรียงไกร วงศ์ใหญ่ | พลตำรวจตรี ว. | `พลตำรวจตรี เกรียงไกร ว.` |
| นายแพทย์ สมชาย ใจดี | นายแพทย์ ใ. | `นพ. สมชาย ใ.` |

กติกาจำแนกยศ: คำลงท้ายด้วยจุด (ร.ต.อ., จ.ส.อ., นพ., ดร., MR.) · ยศสะกดเต็ม
(generate ลำต้น×ตรี/โท/เอก ครอบทหารบก/เรือ/อากาศ/ตำรวจ) · คำขึ้นต้น 'ว่าที่' ·
คำนำหน้าวิชาชีพ/บรรดาศักดิ์ — กันเคส 'นายิกา' (ชื่อขึ้นต้นเหมือนคำนำหน้า) ด้วยการเช็คสระตามหลัง
ทดสอบ 21 เคส + mask ซ้ำ 2 รอบ ผ่านทั้งหมด · compile ผ่าน

หมายเหตุ: snapshot ที่ mask ด้วยโค้ดเก่า ("ร.ต.อ. ม.") กู้ชื่อคืนไม่ได้ —
แต่ session บนเครื่องที่บอร์ดเปิดอยู่มีชื่อเต็ม พอรีสตาร์ท/render ใหม่ จอและ snapshot
จะกลับมาเป็นรูปแบบใหม่เอง (เคสที่ยังอยู่บนบอร์ดวันนี้)

## ข้อมูลที่ค้างอยู่ — จัดการแล้ว/ต้องทำต่อ

✅ **ไฟล์ local** `data/_board_snapshot.json` — mask แล้วทั้ง 13 เคส (ชื่อย่อ + HN 4 ตัวท้าย, `pii_kept=false`) บอร์ดยังกู้ได้ตามปกติ

## เช็กลิสต์ปิดงาน (ฝั่งมุ้กก)

1. รัน `python purge_board_state.py` (ข้างบน) → ต้องเห็น "เหลือ 0"

4. commit **จากเครื่อง Windows เท่านั้น** (มุมมอง WSL/sandbox ตัดท้ายไฟล์ — ดู CODE_REVIEW_PART2 ส่วนที่ 1) และก่อน commit รัน:
   `python -m py_compile main_or_app.py main_or_pages.py main_or_admin.py tracking_board.py purge_board_state.py`
   ```
   git add -A
   git commit -m "security(PII): mask ชื่อ/HN เสมอทุกชั้น (จอ+snapshot+บอร์ดกลาง) — ยกเลิก snapshot_keep_pii + purge script + ล้าง snapshot ที่หลุด (11 มิ.ย.)"
   ```
5. ถ้า 13 เคสเช้านี้เป็น**ผู้ป่วยจริง** (ไม่ใช่ไฟล์ทดสอบ): เก็บไฟล์นี้เป็น incident log — ขอบเขต = ชื่อ+HN+อายุ+diagnosis 13 ราย อยู่บน Supabase ~ไม่กี่ชั่วโมง (10:35 → เวลาที่รัน purge) ผู้เข้าถึงได้ = คนที่มี database_url เท่านั้น (ไม่ใช่ public — RLS + ไม่มี anon key ฝั่งแอป)
